=================================
Decisions (from user): separate package in a monorepo; plugin generates

What shipped
------------
- Converted the repo to an npm-workspaces monorepo:
    packages/graphql-casl/          — the runtime (moved from repo root, unchanged code)
    packages/graphql-casl-codegen/  — NEW: the GraphQL Code Generator plugin
  Root package.json is private, holds shared dev tooling (biome, typescript,
  vitest, coverage, typedoc, semantic-release*) and delegating scripts
  (build/test/typecheck/... fan out via `npm run X --workspaces`).
- Plugin (packages/graphql-casl-codegen/src/index.ts): exports `plugin` + `validate`
  + `GraphqlCaslPluginConfig`. Runs AFTER typescript + typescript-resolvers in the
  same output file; emits (via prepend import + content):
    export type AppSubjectMap = SubjectMap<Resolvers, ResolversTypes>;
    export const Subject = createSubjects<AppSubjectMap>()({ ...object types } as const);
    export const typed = createTyped<AppSubjectMap>();
    export const ability = () => createGraphQLAbility<AppSubjectMap>();
  Excludes root op types + introspection (__*); sorts names; all names configurable.
  createCan stays in app code (needs Context + auth fn) — intentionally NOT generated.
- Tests: packages/graphql-casl-codegen/test/plugin.test.ts (output + config + validate).
- Docs: root README (monorepo overview), packages/graphql-casl-codegen/README.md,
  AGENTS.md rewritten for the monorepo.
- CI: test.yml wiki path -> packages/graphql-casl/docs/api; release.yml matrixes
  over packages and runs semantic-release per package dir.
- .gitignore: docs/api ignore made nested-aware (**/docs/api/).

Verified: build, typecheck, typecheck:tests, test (graphql-casl 33 + codegen 3),
coverage, docs, biome — all green across both workspaces.
